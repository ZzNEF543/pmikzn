<li><strong>JavaScript:</strong> 建议参照 <code>Airbnb JavaScript Style Guide</code> 或 <code>Google JavaScript Style Guide</code>，注重变量声明使用 <code>let</code> 和 <code>const</code> C++ Style Guide</code>，鼓励合理的指针使用和资源管理，禁止魔法数字，要求充分注释和代码尽量模块化分解。</li><br></ul><br><p>遵守这些语言规范，可以极大地改善代码的可读性，并为后续的版本升级和团队协作奠定坚实基础。</p><br><h2 <li><strong>命名规范</strong>：采用统一且语义明确的命名规则。变量、函数、类名称应便于理解业务含义，同时保持简洁。避免使用无意义的缩写和数字，保证代码说明性。</li><br>  <li><strong>代码格式</strong>：严格控制缩进空格数，一般推荐 4 个空格作为缩进标准；适当使用空行分隔逻辑块；避免单行代码过长，超过 80-120 字符时应换行。</li><br>  <li><strong>注释规范</strong>：注释应解释“为什么”而非“做了什么”，保持简短有力。模块、函数、复杂逻辑部分必须有清晰注解。遵守统一的文档注释格式，比如 JavaDoc 或 Docstring。</li><br>  <li><strong>错误处理</strong>：严谨设计异常捕获与处理机制，避免程序崩溃或不可预期状态产生。异常日志清晰，以便后期追踪修复。</li><br>  <li><strong>代码复用</strong>：避免重复编码，提倡抽象并封装重复逻辑，遵守单一职责原则（SRP），保持代码高度模块化与解耦合。</li><br>  <li><strong>安全编码</strong>：防止常见安全漏洞，如SQL注入、XSS攻击等，采用成熟的输入验证和权限控制方案。</li><br></ol><br><p>这些规范不仅帮助程序员写出更规范的代码，也为自动化工具检测提供了标准依据，如静态代码检查、持续集成等。</p><br><h2
